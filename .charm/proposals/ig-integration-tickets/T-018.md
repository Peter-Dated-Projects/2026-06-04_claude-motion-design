## Motivation

The bottom panel and the payoff: shows the generated motion brief and provides the
'Analyze' trigger (Phase B) plus the 'Use as Style Reference' button that delivers
the brief into the main Claude terminal. This is where the extraction becomes usable
for code generation.

## Read first

`.charm/PROJECT.md` (Brief panel + 'Use as Style Reference' bullets + success
criteria 4-5), the proposal's 'Motion Language Brief' + '"Use as Style Reference"
button' + 'Brief -> Code Generation' sections (the exact terminal_input wrapper
text), `CLAUDE.md` (the `terminal_input` command + workspaceStore.setActiveWorkspace
for the workspace switch), and the IG store (T-012) for brief/phase state + the
chosen kept-frame selector + `startPhaseB`.

## Contract this panel binds to (verified)

- Store (T-012, `src/store/igStore.ts`): phase enum is
  `idle | running-A | awaiting-review | running-B | done | error`. This panel renders
  off `phase` plus `brief`, `error`, and the chosen-kept-frame selector. `startPhaseB()`
  takes no args -- it reads the user-overridden kept set internally -- so this panel only
  needs the kept-count (for the disabled/enabled gate) and to call `startPhaseB()`.
- Brief shape (proposal, frozen): `motionLanguage` = { energy, rhythm, pacing,
  transitions, signature }, plus top-level `motionTheme`, `colorMood`,
  `typographyMotion`, `applicationGuide`. Mirror `src/types/ig.ts` (T-012); do not
  re-declare the type here.
- `terminal_input` (verified `src-tauri/src/pty_bridge.rs:307`) is
  `terminal_input(data: String)` -> raw PTY stdin. Call it as
  `invoke("terminal_input", { data })`. There is no other arg; do not touch the PTY.
- Workspace switch (verified `src/store/workspaceStore.ts:95`):
  `useWorkspaceStore.getState().setActiveWorkspace(id)`. The target is the main
  terminal-bearing project workspace. NOTE: today `WORKSPACE_DEFS` holds only the
  `editing` def (`workspaceStore.ts:70`); T-013 (a dependency) adds the IG def and is
  the source of the canonical "main project workspace" id. Reference that id from the
  workspace defs rather than hardcoding a string literal, so this stays correct after
  T-013 lands.

## Scope (owns `src/components/IGWorkspace/BriefPanel.tsx` only)

- Awaiting-review state (`phase === awaiting-review`): an 'Analyze' button (enabled
  only when the chosen-kept-frame count is >=1) that calls the store's `startPhaseB()`;
  show the ~$0.21 cost hint next to it. The button label/cost framing is the explicit
  cost gate for the one paid call -- make the spend legible.
- Analyzing state (`phase === running-B`): a small progress/loading indicator; the
  Analyze button is disabled/replaced so the paid call cannot be double-fired.
- Done state (`phase === done`): the brief rendered as a read-only formatted block --
  the five `motionLanguage` sub-fields, then `motionTheme`, `colorMood`,
  `typographyMotion`, `applicationGuide` -- with the saved `extraction.md` path shown
  beneath (from the store's done-state, not re-derived).
- 'Use as Style Reference' button (appears only when the brief is ready): on click,
  FIRST `setActiveWorkspace(<main project workspace id>)`, THEN
  `invoke("terminal_input", { data })` where `data` is the proposal's exact wrapper:

  ```
  Here is a motion language brief extracted from a reference video.
  Use it to create a Remotion composition with the same motion energy and theme —
  not a copy of the source, but something new that moves and feels the same way.

  Brief: <JSON>
  ```

  `<JSON>` is the stored brief object stringified (`JSON.stringify(brief, null, 2)`).
  This is the one place this panel calls `invoke` directly; pipeline events stay in
  the store.

## Acceptance criteria

- [ ] In `awaiting-review`, 'Analyze' calls `startPhaseB()` and is disabled when the
      chosen kept-frame count is 0; the ~$0.21 cost hint is visible.
- [ ] During `running-B` the Analyze action cannot be fired again (no double-spend).
- [ ] In `done`, the brief renders read-only with all nine fields (five
      `motionLanguage` sub-fields + four top-level) and the saved `extraction.md` path.
- [ ] 'Use as Style Reference' appears only when the brief is ready, switches to the
      main project workspace via `setActiveWorkspace`, THEN sends the exact wrapper
      text (with the brief JSON interpolated) through `invoke("terminal_input", { data })`.
- [ ] Order is switch-then-type, so the typed text lands in a terminal the user is
      already looking at.
- [ ] No PTY / terminal-panel / workspaceStore changes; this panel only consumes their
      existing surfaces.
- [ ] tsc/vite typecheck clean; no `any` for the brief/store reads (use `src/types/ig.ts`).

## Edge cases / notes

- Zero-kept-frames must block Analyze (don't spend on an empty set) -- mirrors the
  backend's zero-frame guard. Drive the gate off the store's chosen-kept selector, not
  the scorer's original kept set, since the user may have rejected every frame.
- Multi-line raw stdin footgun: `terminal_input` writes raw bytes, and the wrapper text
  + pretty JSON contain newlines. Sent naively, an interactive `claude` prompt may
  submit on the first newline instead of receiving the whole block. Wrapping `data` in
  bracketed-paste markers (`ESC[200~` ... `ESC[201~`) is in-scope (it's just bytes in
  the `data` string, no PTY change) and is the likely fix. If end-to-end paste fidelity
  proves to need terminal-side handling, leave it to the integration ticket (T-019) and
  note it -- do not modify the terminal panel here.
- Do NOT auto-submit the prompt (no trailing carriage return that "sends" it). The
  proposal's contract is "exactly as if the user had pasted it themselves" -- pasting
  doesn't submit. Let the user review and hit enter in the terminal.
- A missing/empty brief (analysis `error` phase) shows the store's `error` message,
  not a broken panel or an empty brief block. The 'Use as Style Reference' button must
  not appear in the `error` phase.
- The main-project-workspace id comes from T-013's workspace defs; until T-013 lands,
  reference it through the defs (don't invent a literal). This is the only cross-ticket
  seam -- otherwise the panel is self-contained.
