## Background / motivation

This is the analysis stage of the IG pipeline: turn the kept frames into a
motion-language brief by invoking the real `claude` CLI in print mode. It is
Success criterion 6 in `.charm/PROJECT.md` — the stage that bridges raw frames to
a reusable motion vocabulary. The whole stage hinges on one load-bearing fact
from KB gotcha `claude-cli-no-image-flag`: the CLI has NO `--image` flag (verified
against 2.1.167). Images are attached by listing their absolute paths in the
prompt text and enabling the multimodal `Read` tool, which the agent uses to read
each frame.

Read first (do not duplicate): `.charm/PROJECT.md` (criterion 6, non-goals), the
'Motion Language Extraction' section of `.charm/proposals/PROPOSAL-instagram-pipeline.md`
(the exact prompt + JSON schema, lines 209-284) and `.charm/proposals/SPIKE-instagram-pipeline.md`
(the verified invocation + cost numbers). Import `Brief`, `AnalyzeResult`, the
spawn helper, and `ANALYSIS_MODEL` from T-001 — do not redefine them.

## CRITICAL mechanism (do not deviate)

Build a prompt that lists the kept frames' ABSOLUTE paths, in order, and instructs
the model to Read each, then run exactly:

`echo "$PROMPT" | claude -p --model <ANALYSIS_MODEL> --allowedTools "Read" --output-format json`

- Pipe the prompt on stdin (as the spike did), not as a positional arg — long
  prompts with many paths are cleaner on stdin. Use the T-001 spawn helper.
- `--output-format json` returns a JSON ENVELOPE. Parse it once to get the
  envelope, then JSON-parse the envelope's `.result` STRING a second time to get
  the `Brief`. This double-parse is the easy thing to get wrong.
- Surface `costUsd`, `durationMs`, `numTurns` from the envelope fields into the
  returned `AnalyzeResult` (envelope keys are typically `total_cost_usd`,
  `duration_ms`, `num_turns` — confirm against a live envelope and map to the
  T-001 `AnalyzeResult` field names; do not invent fields).
- The model may wrap `.result` in a markdown code fence (```json ... ```). Strip a
  surrounding fence before the second JSON.parse rather than failing on it.
- Use `ANALYSIS_MODEL` from `lib/constants.ts` — never hardcode the model id here.

## Scope (owns `scripts/ig-pipeline/analyze.ts` only)

- `analyze(keptFramePaths: string[]): Promise<AnalyzeResult>` using the prompt +
  schema from the proposal: `motionLanguage{energy,rhythm,pacing,transitions,signature}`,
  `motionTheme`, `colorMood`, `typographyMotion`, `applicationGuide`. Inject the
  actual frame count for `[N]` in the prompt.
- Validate the parsed brief against the `Brief` shape (all required keys present,
  `motionLanguage` sub-object complete). On malformed/empty/missing-field output,
  throw a clear, actionable error that includes a truncated snippet of what came
  back. Never return a silently-empty or partially-filled brief.
- Thin `import.meta.main` CLI: `bun run analyze.ts <frame paths...>` prints the
  `AnalyzeResult`.

## Acceptance criteria

- [ ] Invokes `claude` in print mode with the frame ABSOLUTE paths listed in the
      prompt and `--allowedTools "Read"` (no `--image` flag anywhere).
- [ ] Parses the JSON envelope, then JSON-parses `.result` into a schema-valid
      `Brief`; returns it as `AnalyzeResult` with `costUsd`/`durationMs`/`numTurns`
      mapped from the envelope.
- [ ] Validates every required `Brief` field including the full `motionLanguage`
      sub-object before returning.
- [ ] Malformed/empty model output yields a clear error with a snippet, not a
      silent empty brief.
- [ ] No live `claude` call runs under `bun test` (see edge cases).
- [ ] Model id comes from `lib/constants.ts`; no inline literal.

## Edge cases / failure modes

- This stage hits the network and costs money (~$0.21, ~40-60s, ~12 turns per run
  for ~11 frames). Do NOT put a live `claude` call in `bun test` — the live path is
  exercised ONLY by the env-gated e2e (T-008). Any unit test here must stub the
  spawn helper / parse a fixture envelope, never shell out.
- Latency scales ~linearly with frame count (one Read round-trip per frame).
- `claude` missing on PATH, a non-zero exit, or empty stdout must each surface a
  distinct actionable error (lean on the T-001 spawn helper's missing-binary vs
  ran-and-failed distinction).
- Envelope present but `.result` is not valid JSON after fence-stripping -> treat
  as malformed output (snippet in the error), not a crash.
- Empty `keptFramePaths` should error before spawning (nothing to analyze), not
  invoke `claude` with a frameless prompt.
- Do NOT implement the `--input-format stream-json` single-turn image optimization
  here — it is an explicit non-goal for v1 (flagged as a follow-up in PROJECT.md
  and the gotcha). Ship the proven Read-tool path.
