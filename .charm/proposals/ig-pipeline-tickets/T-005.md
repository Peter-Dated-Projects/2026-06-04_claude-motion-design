Background / motivation: this is the quality gate of the pipeline — pick sharp, visually distinct, fully-rendered frames and drop static holds, solid-color flashes, and blurry/transition frames, so the analyze stage spends its (paid, multi-turn) Claude budget only on frames worth reading. It is the algorithm-heavy ticket and the only one with fully deterministic, offline tests. Covers success criteria 5 and 7 in `.charm/PROJECT.md`. The spike proved the defaults work untuned: on the test reel, 17 candidates -> 11 kept, 6 rejected (static holds with delta ~1-6 and the blurry tail); sharpness ranged ~48-194.

Read first: `.charm/PROJECT.md`, the 'Frame Selection' and 'Motion Language Extraction' sections of `.charm/proposals/PROPOSAL-instagram-pipeline.md` (the `cv2` snippets there are PSEUDOCODE — reimplement in TS), and `.charm/proposals/SPIKE-instagram-pipeline.md` Finding 4 for reference values. Import every type and constant from T-001 — do NOT redefine them: `ScoredFrame`, `ScoreResult` from `types.ts`; `DELTA_REJECT_THRESHOLD` (=8), `SHARPNESS_FLOOR` (=50), `MAX_KEPT_FRAMES`, `SCALE_WIDTH` from `lib/constants.ts`. Use the image lib already declared in T-001's `package.json` (sharp) — this ticket must NOT add deps or touch `package.json`.

Scope (owns `scripts/ig-pipeline/score.ts`, `scripts/ig-pipeline/score.test.ts`, `scripts/ig-pipeline/fixtures/**` — and nothing else):
- `scoreFrames(framePaths: string[]): Promise<ScoreResult>`. Decode each frame once (via sharp -> raw grayscale buffer for sharpness/delta, raw RGB buffer for entropy) and compute three metrics:
  - Sharpness = variance of the Laplacian over the grayscale buffer. Use the standard 3x3 Laplacian kernel `[[0,1,0],[1,-4,1],[0,1,0]]`; document it in a comment. Sharpness is the variance of the convolved response (not its mean).
  - Delta = mean of the absolute per-pixel grayscale difference vs the PREVIOUS frame in input order. The first frame has no predecessor: record `delta` as null/undefined and never reject it on delta.
  - Entropy = Shannon entropy (log2, in bits) of a normalized RGB color histogram. Resolve the binning explicitly (see edge cases) and document it; the exact bin scheme changes the numbers and must match what fixtures pin.
- Filter + rank: reject `delta < DELTA_REJECT_THRESHOLD` (first frame exempt), then reject `sharpness < SHARPNESS_FLOOR`; rank the survivors by the combined score `sharpness * 0.7 + normalizedEntropy * 0.3` (per the proposal) and keep the top `MAX_KEPT_FRAMES`. Each `ScoredFrame` records all three raw metrics, its `kept` flag, and a `rejectReason` (`"low-delta"` | `"low-sharpness"` | `"below-top-N"`) when not kept. `ScoreResult` returns all scored frames plus the kept set in ranked order.
- Warn (stderr, non-fatal) if fewer than 5 frames survive all filters — matches the proposal's "too short / too fast-cut for reliable analysis" guidance.
- Thin `import.meta.main` CLI: `bun run score.ts <frames-dir-or-paths...>` resolves a directory to its sorted frame list or takes explicit paths, runs `scoreFrames`, and prints the `ScoreResult` as a readable table (path, sharpness, delta, entropy, kept, reason).

Tests (`bun test`, fully offline — no network, no ffmpeg, no `claude`): commit a SMALL fixture set under `fixtures/` — synthetic images with analytically KNOWN sharpness/delta/entropy are preferred over real frames because they pin exact values and stay commit-tiny (e.g. a flat solid -> entropy ~0 and sharpness ~0; a sharp checkerboard -> high Laplacian variance; an A/B pair with a known mean-abs difference). Assert each metric function against its expected value within a stated tolerance, and assert the end-to-end keep/reject/rank decisions on a crafted set (at least one low-delta reject, one low-sharpness reject, one below-top-N reject, and the first-frame-never-rejected-on-delta case).

Acceptance criteria:
- [ ] `bun test` passes offline using only committed fixtures; no network, ffmpeg, or claude invoked.
- [ ] Each metric function (sharpness, delta, entropy) matches its expected fixture value within a documented tolerance.
- [ ] First frame is never rejected for low delta; its delta is recorded as null/undefined, not 0.
- [ ] A crafted fixture set exercises all three reject reasons (low-delta, low-sharpness, below-top-N) and survivors come back ranked by the combined score.
- [ ] `< 5` survivors emits the warning (asserted in a test) without throwing.
- [ ] All thresholds and weights trace to `lib/constants.ts` (overridable); zero magic numbers inlined in `score.ts`.
- [ ] CLI runs: `bun run score.ts <dir>` prints the table and exits 0; this ticket adds no dependency and does not edit `package.json`.

Edge cases / failure modes:
- ENTROPY BINNING (resolve before writing fixtures): the proposal's `cv2.calcHist([0,1,2], [8,8,8])` is a single JOINT 3D histogram of 512 bins (8 levels per channel), and entropy is computed over that — NOT three separate per-channel 8-bin histograms summed. The original ticket text said "8 bins/channel," which is ambiguous; implement the joint 8x8x8 scheme to match the proposal/spike reference, and pin fixtures against THAT. Document the chosen scheme in a comment so the pinned values are self-explanatory.
- `normalizedEntropy` in the combined score must be on a comparable scale to raw sharpness before the 0.7/0.3 weighting; define and document the normalization (e.g. divide by max possible entropy = log2(occupied-able bin count)) rather than mixing a ~0-9 entropy with a ~0-300 sharpness raw.
- Laplacian at image borders: pick and document a border policy (clamp/replicate or skip the 1px border) — it must be consistent so fixture values are reproducible across runs/platforms.
- sharp returns raw pixel buffers in a known channel order/stride; do the grayscale conversion once and reuse it, and handle RGBA vs RGB stride, or sharpness/delta will be subtly wrong.
- Keep fixtures tiny (small dimensions, PNG) so the repo stays light; prefer generating them in-test from buffers if that keeps values exact and avoids committing binaries.
- Avoid OpenCV entirely (no Python 3.14 wheels; unnecessary) — all math in TS over sharp buffers.

Note on `MAX_KEPT_FRAMES`: the proposal text says "top 30" but the spike kept 11 of 17 and PROJECT/T-001 make this a named constant. Honor the T-001 constant value; do not inline 30.
